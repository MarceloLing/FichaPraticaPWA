Perguntas e Respostas

  1. O que são operações de Commit, Push e Pop efetuadas num repositório?

    2. R: As operações acima mencionadas são de extrema importância pois elas serão usadas para o manuseio dos ficheiros a serem submetidos no GitHub.

    • A Womakerscode (2023), define o Commit , como um grupo de alterações no código, contendo as alterações que foram feitas nele e uma mensagem descritiva, além de informações meta (data, autor, etc).
    • A Womakerscode (2023), define o Push como um comando usado para enviar dados para o repositório remoto.
    • A Womakerscode (2023), define o Pop como um comando que irá restaurar alterações salvas temporariamente e removê-las da área de armazenamento.

  2. Quais são as vantagens de usar Branchs num repositório git?

    3. R: De acordo a Womakerscode (2023), na sua definição diz que Branches são separações de código e que estes normalmente são utilizados para separar alterações grandes ou novas funcionalidades de um projeto, sendo assim essa funcionalidade cria múltiplas vantagens para quem tem um grande projeto em produção, proporcionando segurança, estabilidade e uma maior abertura na implementação de novas funcionalidade e ou outros serviços sem comprometer o código principal, ou seja ele cria ramificações do main code.

  3. Nos Merges de conteúdos resultantes de um Pop podem resultar em conflitos. Em que consistem e como podem ser resolvidos esses conflitos?

    4. R: De acordo ao GitHub Docs , os merges resultantes de um Pop podem causar conflitos que consistem em conflitos de conteúdo ou de arquivo, para resolução desses conflitos podemos recorrrer a uma avaliação de cada ficheiro onde o Git irá previamente identificar as diferenças, escolher como lidar com as diferenças para posterior proceder com um merge sem erros

  4. Clone e Fork são formas diferentes de replicar um repositório Git. Quais as diferenças entre eles?

    5. R: De acordo com a Womakerscode (2023), Um Clone de um repositório funciona como uma branch de um repositório online em um repositório local. Ou seja, quando se deseja trabalhar em um repositório hospedado no github, clona-se esse repositório para o seu computador, trabalha-se nele, e então é pedida a permissão para atualizar as alterações online. já o Fork é como um clone, porém dentro do github. Isso quer dizer que o repositório não vai ser baixado para seu computador, mas será criado um igual na sua conta.
